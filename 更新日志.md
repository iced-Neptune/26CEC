# 更新日志

## [2026-04-22] v1.0 - 合并增强版功能，完成架构重组与注释规范化

### 新增
- **UI 交互增强**
  - 在 `monitor_ui.py` 中新增历史回看时间轴滑块控件及回调 `on_slider_move`。
  - 新增光标十字线与悬浮气泡功能（`on_mouse_move` 增强），提升数据查看体验。
  - 控制面板新增“允许接收传感器数据”复选框，替代原有的“显示波形”开关。
- **数据管理**
  - 在 `serial_handler.py` 中新增 `start_json_recording` 和 `stop_json_recording` 方法，支持手动控制 JSON 录制。
  - 引入 `pending_json_state` 状态机，精确标记 JSON 数据点的事件类型（反应开始/结束）。
- **绘图模块**
  - `plot_manager.py` 支持多图层管理（基准线、阈值线、起始/结束竖线、十字线）。
  - 坐标轴管理支持“动态滚动/冻结/滑块回看”三种模式。

### 变更
- **模块拆分重组**：将单文件新功能代码按原项目架构拆分为 UI、Core、Utils 模块，保持代码高内聚低耦合。
- **注释规范化**：所有新函数及修改函数均添加了符合 Google Python Style Guide 的注释块，语言采用口语化类比描述，并标注置信度。
- **UI 布局调整**：右侧面板重新组织为“核心逻辑侦测台”和“光电特征追踪”，信息展示更清晰。

### 修复
- 修复了 JSON 录制过程中状态标签不准确的问题。
- 修复了未选择 JSON 保存目录时程序可能崩溃的问题（增加默认目录回退）。
- 修复了绘图线程在连接断开后仍持续运行导致资源占用的问题（通过 `is_connected` 标志控制）。

### 规范审查备注
- 所有代码块均已通过 `pylint` 基础检查，无明显语法错误或未使用导入。
- 新增功能均通过手动测试（模拟数据）验证可用性。

### 文件差异统计
- 修改文件：`src/main.py`、`src/ui/monitor_ui.py`、`src/core/data_processor.py`、`src/core/serial_handler.py`、`src/core/plot_manager.py`、`src/utils/file_dialog.py`
- 新增文件：无（所有功能嵌入现有模块）
- 删除文件：无

### 后续建议
- 若需要版本控制，建议创建 Git 提交并撰写详细的提交信息。
- 可考虑添加 `requirements.txt` 文件明确依赖版本（`matplotlib>=3.7.0`, `numpy>=1.24.0`, `pyserial>=3.5`）。

## [2026-05-07] v1.1 - 项目结构规范化并适配单文件 exe 打包，优化输出目录与体积

### 新增功能
- 无

### 变更功能
- **项目结构重构**：将入口 `main.py` 提升至项目根目录，源码包统一为 `src/`，更符合标准 Python 项目布局。
- **路径机制重构**：新增 `path_helper.py` 工具模块，使程序自动识别 exe 所在目录，配置文件与数据文件均相对于该目录读写，兼容开发与打包环境。
- **输出目录优化**：CSV 文件默认保存路径调整为 `data/csv_records/`，JSON 文件默认保存路径调整为 `data/json_raw/`，文件对话框默认指向对应子目录。
- **打包策略变更**：由文件夹模式改为 **单文件模式（`--onefile`）**，并强制 `matplotlib` 使用 `TkAgg` 后端，彻底排除 Qt 依赖，打包体积减少约 90%。

### 修复功能
- 无

### 其他更新
- 移除了因 Conda 环境引入的多余 Qt 库（PySide6/shiboken6），超大体积GCK。
- 所有配置与数据目录均支持自动创建，首次启动无需手动准备。
- 更新了打包命令pyinstaller --onefile --name "我想去看音律联觉" --exclude-module PySide6 --exclude-module shiboken6 main.py
