一、上下位机功能总结
1.1 下位机功能（Arduino）
1.1.1 传感器数据采集与滤波
通过模拟引脚以10ms周期读取光敏传感器原始值，采用5点滑动平均滤波算法抑制噪声干扰，输出稳定光强数据。

1.1.2 动态环境光标定
系统启动后自动进入标定状态，连续采集3秒内稳定光强数据，计算算术平均值作为环境基准线，并依据预设百分比计算反应开始阈值与结束阈值。

1.1.3 反应过程状态机控制
根据光强变化触发状态切换，依次执行反应启动确认、保护期计时、小车电机驱动、结束判定确认等动作，同时通过蜂鸣器发出提示音。

1.1.4 串口通信与事件上报
以115200波特率向上位机发送实时光强数值，并在状态切换时输出特定事件标识（如WAITING_REACTION、REACTION_START、REACTION_TIME、MEASUREMENT_COMPLETE等），支持接收RESET指令重置系统。

1.2 上位机功能（Python）
1.2.1 串口数据接收与解析
通过pyserial库建立与下位机的串行通信，实时接收光强数据流并解析下位机发出的状态事件字符串，更新界面显示与内部状态变量。

1.2.2 实时波形绘制与交互
使用matplotlib绘制光强随时间变化曲线，支持动态滚动显示、光标悬停数值查询、图像冻结功能，并在图中标注环境基准线、触发阈值线、反应开始与结束时刻线。

1.2.3 实验数据记录与导出
自动追踪反应过程关键参数（环境基准光强、起始光强、平均光强、最低光强、结束光强、反应耗时等），将每次实验汇总数据追加至CSV表格文件，同时将单次反应的全部原始采样点及其对应时间与状态标签以JSON格式独立保存。

1.2.4 测量控制与状态显示
提供连接管理、端口刷新、测量重置等交互控件，在界面右侧显示当前实验状态、反应耗时及实时光强追踪数据（环境基准、当前值、平均值、最低值）。

二、反应开始的判断逻辑
2.1 环境基准建立
系统上电后首先进入标定状态，在确认光强高于800的条件下持续采集3秒内第500毫秒后的数据点，计算平均值作为环境基准光强（baseline_light）。该基准值代表溶液加入前的背景光水平。

2.2 开始阈值计算
根据预设常量START_PERCENT（默认值0.15），计算开始阈值：
start_threshold = baseline_light × (1 - START_PERCENT)
即光强需较环境基准下降15%以上方视为溶液加入。

2.3 下降确认与抗抖动
标定完成后状态机进入等待状态（state=1），持续监测滤波后光强值。当光强低于开始阈值时，启动连续计数稳定判定：由于采样周期为10ms，需连续20次采样均低于阈值（累计200ms）才确认为有效触发，滤除因飞虫、瞬时遮挡等造成的短时波动。确认后立即记录开始时刻（start_time），发出提示音，向上位机发送REACTION_START事件，状态机转入反应进行中状态（state=2）。

2.4 保护期机制
反应开始后设置10秒保护期（PROTECTION_TIME），在此期间即使光强再次低于结束阈值也不进行结束判定，避免反应初期因混合不均匀导致的误判。

三、反应结束的判断逻辑
3.1 结束阈值计算
结束阈值依据预设常量END_PERCENT（默认值0.25）计算：
end_threshold = baseline_light × (1 - END_PERCENT)
即光强较环境基准下降25%时认为反应已达终点（“见好就收”策略）。

3.2 阈值穿越检测
反应进入状态2后，当保护期结束（now - start_time > PROTECTION_TIME），系统开始监测光强是否低于end_threshold。一旦满足条件，记录当前时间为confirm_start，向上位机发送END_DETECTED事件，状态机进入确认期（state=3）。

3.3 确认期抗误判滤波
在确认期内持续采样800ms（CONFIRM_TIME）。统计该时段内光强低于end_threshold × 1.1 的采样次数（允许10%容差以应对轻微波动）。若有效低点数量超过总采样数的一半，则判定为真实变色反应结束；否则视为误触发，状态回退至state=2继续监测。

3.4 结束处理与数据输出
确认结束后状态机转入state=4，延时1秒（CAR_DELAY）后关闭小车电机，向上位机发送反应耗时（REACTION_TIME:xxxms）及测量完成（MEASUREMENT_COMPLETE）事件，同时通过串口向串口屏发送反应秒数数据。状态机最终进入闲置状态（state=5），等待RESET指令重置。
